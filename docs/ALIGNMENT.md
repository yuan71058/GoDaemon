## 一、原始需求

### 1.1 需求来源
基于技术文档《Go语言 类大漠插件（后台版）开发技术栈完整MD文档（详细版）》开发一套类大漠插件的Go语言后台自动化框架。

### 1.2 核心需求
开发一个开源、高性能、纯后台的Windows自动化框架，功能对标大漠插件v7.1908核心功能：

1. **窗口后台绑定** - 支持多种绑定模式（dx2、dx3、gdi模式）
2. **后台截图** - 窗口最小化、被遮挡时仍可正常截图
3. **后台键鼠操作** - 不抢占前台，不影响正常电脑使用
4. **图色识别** - 找图、找色、多点比色，支持相似度调节
5. **文字OCR识别** - 本地离线识别，支持中文、游戏字体
6. **DLL导出** - 供易语言、Python、C#、按键精灵等跨语言调用

## 二、需求边界

### 2.1 包含范围
| 功能 | 详细说明 | 优先级 |
|------|----------|--------|
| 窗口管理 | 句柄获取、窗口绑定/解绑、状态判断、尺寸获取 | P0 |
| 后台截图 | GetWindowDC+BitBlt实现，支持PNG/JPG/BMP格式 | P0 |
| 键鼠操作 | PostMessage/SendMessage实现后台操作 | P0 |
| 找图功能 | gocv模板匹配，支持相似度调节 | P0 |
| 找色功能 | RGB/HSV颜色模式，支持容差调节 | P0 |
| OCR识别 | Tesseract本地离线识别 | P1 |
| DLL导出 | 64位DLL优先，32位可选 | P0 |
| 多点比色 | 坐标组批量校验 | P1 |
| 旋转找图 | ±15°旋转找图 | P2 |

### 2.2 不包含范围
- 驱动级键鼠操作（可作为后续扩展）
- 网络OCR服务（仅本地离线识别）
- 多窗口并行管理（可后续扩展）
- 游戏反绑过绑（超出基础功能范围）

## 三、技术约束

### 3.1 开发环境
| 项目 | 版本要求 | 说明 |
|------|----------|------|
| Go | 1.21+ | 原生支持DLL编译 |
| MinGW-w64 | 12.2.0 | CGO编译依赖 |
| OpenCV | 4.8.0 | gocv依赖 |
| Tesseract | 5.3.0 | OCR引擎 |

### 3.2 目标平台
- **操作系统**: Windows 7/10/11 (64位优先)
- **输出格式**: 64位DLL（damo64.dll）、32位DLL（damo32.dll）

### 3.3 依赖约束
- 所有依赖必须开源可商用
- 不使用闭源DLL（如ScreenCapture.dll、ImageSearch.dll）
- 不使用钩子（HOOK）、不注入进程
